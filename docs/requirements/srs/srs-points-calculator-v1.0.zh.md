# 软件要求规格说明书（SRS）

**项目名称**: 点数换算小工具
**版本**: 1.0
**日期**: 2026-04-09
**作者**: Requirements Analyst AI

---

## 1. 简介

### 1.1 目的
本文档定义了“点数换算小工具”的软件要求。

### 1.2 范围
- **范围内的功能**: 一个供内部运营人员和外部客户使用、快速计算点数配置及预估大型语言模型（LLM）API 费用的独立前端工具，计算基于特定公式。
- **范围外的功能**: 后端集成、用户认证、数据持久化。

### 1.3 定义和缩写
- **Tokens**: LLM 处理数据的基本单位（例如：输入 tokens、输出 tokens）。
- **点数 (Points)**: 系统内表示 API 使用成本的计算货币或度量单位。
- **公式**: `点数 = (((输入 tokens / 1000) * 输入单价 + (输出 tokens / 1000) * 输出单价) * 损耗率 * 利润率) / 点数系数`。

---

## 2. 系统概述

### 2.1 系统目的
提供一个快速、易用的计算器，用于预估 API 成本并将其转换为标准化的“点数”指标，主要服务于内部运营人员和外部客户。

### 2.2 用户
- **内部运营人员**: 用于快速计算点数配置。（预计：< 100 名用户）
- **外部客户（终端用户/开发者）**: 用于预估 LLM API 费用。（预计：< 100 名用户）

### 2.3 目标环境
- **浏览器**: 现代 Web 浏览器（Chrome、Firefox、Safari、Edge）。
- **设备**: 桌面、平板、智能手机（响应式设计）。
- **网络**: 初始加载需连接互联网；之后可纯在浏览器内运行。

---

## 3. 功能要求

### 3.1 模型选择与单价获取
- FR-001: 系统应提供一个可用 LLM 模型的下拉列表。
- FR-002: 系统应允许用户手动输入进行模型名称的模糊搜索。
- FR-003: 在选择一个模型后，系统应自动填充对应的“输入单价”和“输出单价”字段。
- FR-004: 填充后的单价字段应允许用户修改。

### 3.2 参数输入
- FR-011: 系统应提供以下参数的输入字段：输入 Tokens、输出 Tokens、输入单价、输出单价、损耗率、利润率和点数系数。
- FR-012: 系统应在初始化时为特定字段填充默认值：损耗率 (20%)、利润率 (50%)、点数系数 (0.02)。

### 3.3 实时点数计算
- FR-021: 当任何参数被输入或修改时，系统应自动、实时地计算总“点数”。
- FR-022: 计算必须严格遵守公式：`点数 = (((输入 tokens / 1000) * 输入单价 + (输出 tokens / 1000) * 输出单价) * 损耗率 * 利润率) / 点数系数`。
- FR-023: 系统应在页面上清晰地显示最终计算出的“点数”数值。
- FR-024: 系统应（可选地）显示计算的中间明细，以便用户核对。

---

## 4. 非功能要求

### 4.1 性能
- NFR-001: 系统应具有快速的响应时间，在参数修改后几乎瞬间（例如 < 0.5 秒）显示计算结果。

### 4.2 安全性
- NFR-011: 作为一个无后端的纯前端工具，不需要特定的安全要求（如认证、授权）。

### 4.3 可用性
- NFR-021: 没有严格的高可用性要求；作为一个内部工具，短时间的不可用是可以接受的。

### 4.4 可扩展性
- NFR-031: 代码库的设计应易于适应未来添加更复杂的计费公式和扩展模型列表的需求。

---

## 5. 外部接口

### 5.1 用户界面
- 一个干净、直观、响应式的 Web 界面，适合快速计算。

### 5.2 软件接口
- 无（独立的纯前端应用）。

---

## 6. 系统特性

### 6.1 易用性
- 界面必须高度可用和直观，允许用户在无需事先培训的情况下进行计算。
